## 语义图谱

### 核心思路
- 通过 LLM 分析文件内容，自动发现没有 [[链接]] 但语义相关的文件关联
- 补充图谱的边，让知识图谱更丰富

### 流程
- 用户点击"分析语义关联"按钮触发（手动触发，因为 LLM 调用有成本）
- 收集所有 .md 文件的结构化摘要（不是截取前 200 字）
- 将摘要批量发送给 LLM，请求分析文件间的语义关联
- LLM 返回语义边列表：{ source, target, reason, strength }
- 语义边存入 .vision/semantic-edges.json
- 图谱中语义边用虚线显示，hover 显示 reason
- 用户可以确认或删除某条语义边

### 内容摘要策略讨论
- 截取前 200 字不够充分，可能只覆盖开头/引言
- 更好的方案：分层摘要
  - 标题 + 小标题（heading）— 文章结构骨架，信息密度高
  - 首尾段落 — 开头说明目的/背景，结尾总结/结论
  - 关键词提取 — 全文高频专业术语/概念词（简单算法，不需要 LLM）
  - 短文件（<500字）直接用全文；长文件用组合摘要
- 待定：用哪个 LLM 做分析？当前配置的 Agent model vs 专门的轻量模型（Haiku）

### 增量更新
- 文件变更时只重新分析变更的文件，不全量重跑

### 可审核
- 用户可以查看每条语义边的 reason，删除不合理的

## Skill 市场 / 管理

### 核心思路
- 用户可以浏览、搜索、安装、卸载 Skill
- 类似 VS Code 扩展市场的体验

### 功能规划
- Skill 市场：浏览官方和社区 Skill，查看描述、示例、评分
- 一键安装 / 卸载
- Skill 管理面板：查看已安装 Skill、启用/禁用、配置参数
- 用户自定义 Skill：通过 UI 创建和编辑（不需要手写 md 文件放到目录）
- Skill 版本更新提示

## MCP Server 管理

### 核心思路
- 用户可以配置、连接、管理 MCP 服务器，扩展 Agent 能力

### 功能规划
- MCP 配置 UI：在 Settings 中添加 MCP Server 管理区域（名称、命令、参数、环境变量）
- 配置持久化：存储到 store，Agent 启动时传入 `Options.mcpServers`
- MCP 状态显示：Agent 运行时展示各服务器连接状态（通过 SDK `Query.mcpServerStatus`）
- MCP 工具权限：MCP 工具以 `mcp__<serverName>__<toolName>` 格式出现在权限控制中
- 预置 MCP 服务器模板（如 filesystem、github 等），降低配置门槛
